🔧 Crear Buckets de Storage

Sistema automático para crear storage en Supabase

⚠️ Importante: Esta página se conecta directamente a Supabase. Para crear buckets necesitas que tu usuario tenga permisos de service_role o ser dueño del proyecto. La clave anon solo puede listar/verificar buckets existentes.
📋 Qué va a pasar:
✅ Se crearán 4 buckets:
• reports (PDFs de reportes de inspección — público)
• report-evidencias (fotos y firmas de inspección — privado)
• fauna_impact_pdfs (para PDFs de impactos — privado)
• fauna_rescue_pdfs (para PDFs de rescates — privado)

✅ Se verificará que existen en Supabase
✅ Se mostrará el resultado en la consola
⏳ Listo para ejecutar
🔵 Iniciando...
📝 Próximo paso:
Después de crear los buckets, ve a:
Supabase Dashboard → Storage → [bucket] → Policies

Para report-evidencias: agrega políticas INSERT y SELECT para usuarios autenticados.
Para reports: bucket público, sin cambios necesarios.
Para fauna_impact_pdfs y fauna_rescue_pdfs: agrega políticas SELECT, INSERT, DELETE.

Ver instrucciones en CREATE_BUCKETS.sql
